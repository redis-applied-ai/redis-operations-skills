# Redis Software DNS Delegation

Use this skill when planning or debugging Redis Software DNS for cluster and database access.

## Key Rules

- The Redis Software cluster FQDN is chosen during installation and should be treated as permanent.
- The FQDN must use DNS-safe labels: letters, numbers, and hyphens; no underscores; each label starts and ends with a letter or number.
- Redis Software uses the cluster FQDN to generate database endpoint names such as `redis-<port>.<cluster-fqdn>`.
- Redis Software runs its own bundled PowerDNS authority on cluster nodes; do not instruct users to install PowerDNS manually.
- Load balancer access is a fallback when delegation is impossible and can reduce transparent failover or migration behavior.

## Setup Workflow

1. Choose a subdomain for the cluster, for example `mycluster.example.com`.
2. Confirm the parent DNS zone is controlled by the user or their network team.
3. During Redis Software installation, set the cluster name/FQDN to the chosen subdomain.
4. In the parent DNS zone, delegate the cluster subdomain to Redis cluster nodes with NS records.
5. Add glue A records mapping each node hostname under the cluster FQDN to its node IP.
6. Wait for DNS propagation.
7. Verify a generated database endpoint resolves from client networks.

## Required Record Pattern

Use this shape, replacing names and IPs:

```dns
<cluster-fqdn>.       IN NS <node1>.<cluster-fqdn>.
<cluster-fqdn>.       IN NS <node2>.<cluster-fqdn>.
<cluster-fqdn>.       IN NS <node3>.<cluster-fqdn>.

<node1>.<cluster-fqdn>. IN A <node1-ip>
<node2>.<cluster-fqdn>. IN A <node2-ip>
<node3>.<cluster-fqdn>. IN A <node3-ip>
```

Database endpoints are then generated by Redis Software, usually as:

```text
redis-<port>.<cluster-fqdn>
```

Do not add separate DNS records for each database endpoint unless official guidance for the specific deployment says otherwise.

## Verification Commands

Check endpoint resolution:

```bash
dig redis-<port>.<cluster-fqdn>
host redis-<port>.<cluster-fqdn>
nslookup redis-<port>.<cluster-fqdn>
```

Check DNS reachability to a node:

```bash
nc -vzu <node-ip> 53
```

If `nc` fails, inspect firewalls, security groups, network ACLs, and local host firewall rules for UDP port 53.

## Troubleshooting

| Symptom | Check |
| --- | --- |
| Database endpoint does not resolve | Parent zone delegation, NS records, glue A records, propagation, and resolver path. |
| Resolution works on a node but not from clients | Client network cannot reach node DNS service or uses a resolver that cannot follow delegation. |
| Failover does not update quickly | Check resolver caching; Redis database DNS records are intended to use short TTLs. |
| Install rejects cluster name | FQDN label has invalid characters, starts/ends with a hyphen, or includes underscores. |
| UDP 53 is blocked | Firewalls or security groups block DNS queries to cluster nodes. |
| User asks for reverse DNS | Redis Software does not require PTR records for this setup. |

## Escalation Packet

Collect:

- Cluster FQDN chosen during installation.
- Parent DNS zone and DNS provider.
- Node hostnames and IP addresses.
- Configured NS and glue A records.
- Example database endpoint.
- `dig` or `host` output from both inside and outside the client network.
- UDP 53 test results to each node.
- Whether a load balancer fallback is being used.
